1. Buatkan contoh data JSON dengan mengacu pada format class @AggregateAndInsertPayloadModel . Untuk value code gunakan prefix random aja misal MAP0001, atau AGG0001.  
2. Buatkan kode untuk memproses data dari data json tersebut dengan kriteria sebagai berikut:
    Kriteria:

    2.1. C#:
        Gunakan bahasa c#.

    2.2. data:
        - Objek JSON berisi data untuk proses aggregasi (MAX, MIN, COUNT, AVG, SUM).
        - Mendukung operasi aritmatika (+, -, *, /), termasuk operasi berantai.
        - Preseden operator dan tanda kurung harus diperhatikan.
        - Mendukung kondisi (WHERE, OR/AND, IN, LIKE, BETWEEN).
        - Mendukung ordering ASC/DESC.
        - Struktur bebas (misalnya organisasi, transaksi, inventaris).

    2.3. Path JSON:
        - Gunakan notasi array [*] untuk iterasi semua elemen.
        - Mendukung single element [0] atau multiple [0,2,4,5...].
        - Contoh path:
            - data.department.teams.employee.salary
            - data.department.teams.employee[*].salary

    2.4. Kombinasi Dinamis:
        - Data bisa berupa raw atau hasil aggregasi.
        - Bisa dikombinasikan dengan filter dan order.
        - Mendukung kombinasi fungsi aggregasi dengan operasi aritmatika.
        - Mendukung query DSL dengan separator "~". element pertama untuk pengambilan data, elemen kedua untuk filtrasi, dan elemen ketiga untuk ordering. Lihat point 7 untuk contoh referensi format querynya

    2.5. Implementasi C#:
        - Buat minimal 5 variasi implementasi untuk masing-masing kasus berikut:
            - Aggregasi
            - Operasi aritmatika
            - Filter data
            - Ordering data
            - Kombinasi aggregasi + aritmatika + filter + order
            - Preseden operator
            - Preseden tanda kurung
        - Gunakan `Newtonsoft` untuk proses pengolahan Json/JObject.
        - Tidak perlu membuat model (data dinamis).
        - Gunakan path JSON DSL, bukan LINQ.
        - Hindari warning IDE/CS/SonarQube.
        - Perhatikan nullable vs non-nullable.
        - Tidak perlu unit test.

    2.6. Tambahan:
        - Sertakan variasi struktur JSON (organisasi, transaksi, inventaris).
        - Gunakan nilai numerik dan boolean realistis.
        - Pastikan path JSON konsisten.
        - Kode rapih, efisien, performa tinggi.

    2.7. Referensi format query:
        - "SUM(data.organization.departments[*].teams[*].employees[*].salary)"
        - "AVG(data.organization.departments[*].teams[*].employees[*].salary~data.organization.departments[*].teams[*].employees[*].isActive = true")"
        - "SUM(data.organization.departments[0].teams[*].employees[*].salary) / COUNT(data.organization.departments[0].teams[*].employees[*])"
        - "SUM(data.organization.inventory[*].price * data.organization.inventory[*].stock)"
        - "4 + 5 * 4 / 2 - 3"
        - "4 + (5 * (4 / 2)) - 3

    Catatan:
    - Buat satu class khusus dengan nama `AggregationProcessor` untuk handling semua proses.
    - Isi data pada model @AggregateAndInsertPayloadModel bersifat dinamis, jadi jangan pernah menyebutkan nama field secara hardcode.
    - Buatkan extension untuk @AggregateAndInsertPayloadModel untuk memproses setiap json property pada field "data" di model tersebut, dengan output JOBject:
        {
            "fieldA": "{aggregation result of fieldA}",
            "fieldB": "{aggregation result of fieldB}",
            ...
        }
    - Berikan kode secara lengkap, dan cukup letakkan di satu file dengan nama AggregationProcessor
